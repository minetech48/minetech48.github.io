This is a software Mandelbrot renderer written to use Javaâ€™s Swing library. The program splits the frame up into segments in order to allow for use of multiple CPU threads.
A better explanation of the Mandelbrot set than I could possibly do can be found on wikipedia.

Figure 2 is an image taken on a zoomed in part of the fractal. the colors corespond to the number of complex itterations before Z diverges to infinity (>2)

This program implements the complex equation in Java on the CPU, and renders it using the Swing library. It has some built in keybindings to increment or decrement the number of itterations it calculates. It also copies and scales the previous frame into the current one for more seamless use. Finally, this algorithm is highly parallelizable, and while this program runs it on the CPU, it is made to split up the screen in order to make use of multiple CPU cores (figure 1).



This program uses 64-bit floating point numbers for its calculations, and thus, runs into precision errors around a scale of 10^17 (figure 3). This could be solved by using larger or even dynamically sized floating point numbers.
